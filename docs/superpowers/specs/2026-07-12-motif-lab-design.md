## 1. 一句话

用**母题故事动画**把难概念演明白：故事在讲，原理在演；孩子与成人共用一套极简界面。

## 2. 背景与决策摘要

| 决策 | 选择 | 原因 |
|------|------|------|
| 与 Tepeu 关系 | **新项目**，不改造 Tepeu 本体 | 学习演示 ≠ 编程 IDE；避免两套产品互拖 |
| 实现策略 | **薄壳**：只借对话流、HTML 预览、工作区/密钥思路 | 产品干净，迭代快 |
| 受众 | 孩子 + 自己，**同一套极简 UI** | 不做双皮肤 |
| 长期学科 | 数理 / 科学 / 编程 / 现问现做 **全要** | 分期；共用「对话 + 动画舞台」底座 |
| 首版切口 | **编程概念**（先打磨「循环」） | 验证「故事↔原理」动画是否成立 |
| 动画策略（体验） | **默认就有动画**；不懂再出更简版 | 对齐母题「教」；`/重写` 动画版 |
| 动画策略（工程） | **骨架秒出 + 缓存命中秒播 + 无缓存可等待** | 避免 C 策略下的干等白屏 |

**明确否定的方向**：以预置题库点选为**主路径**；先纯文字、不懂再甩无关演示；Fork 整仓 Tepeu 改皮。

> 注：内置「循环」金牌动画是**缓存种子**，不是「题库点选学习」产品形态。

## 3. 用户与成功标准

### 3.1 用户
- **孩子**：听故事、看动画、答简单检验；少打字，可点「不懂 / 更简单」。
- **成人（你）**：同一界面自学难概念；可输入任意概念，**v0.1 仅承诺「循环」质量**，其余为试做。

### 3.2 v0.1 成功标准（可测）
1. 输入概念后，**3 秒内**右侧或对话区出现**故事分镜骨架**（非白屏）。
2. **缓存命中**时动画立即可播；未命中时显示生成进度，完成后自动播放。
3. 「不懂 / 更简单」能再生成一版**要素更少**的动画（非同一文件重播）。
4. 学完「循环」后，能通过界面 **1～2 道检验题**（主题：它在重复做什么）。

## 4. 信息架构与界面

### 4.1 一屏两栏
```
┌─────────────────────┬──────────────────────┐
│ 左：母题对话         │ 右：动画舞台          │
│ · 概念 / 角色提示    │ · 分镜骨架（先）      │
│ · 故事口诀检验       │ · HTML/Canvas 播放   │
│ · 「不懂」「更简单」  │ · 进度 / 重生成      │
└─────────────────────┴──────────────────────┘
```

### 4.2 不做（留给 Tepeu）
文件树、终端、多技能市场、IDE 三栏、通用 Agent 改仓库代码。

### 4.3 视觉语气
极简、字号偏大、少装饰；跟系统亮暗即可。不以「儿童卡通皮肤」和「极客 IDE」分裂成两套。

## 5. 学习流（母题加强版）

对齐 `learn-skill` **母题导师**四步；Step 1 升级为「故事 + 动画一体」。

| 步 | 母题原意 | Motif Lab 形态 |
|----|----------|----------------|
| 1 教 | 母题寓言（物理悖论 + 人物动作） | **先骨架 → 再动画**；动画同时讲故事与演原理 |
| 2 学 | 80/20 血清（数字/口诀） | 动画就绪后对话区 **1 句口诀**（v0.1 必做） |
| 3 检验 | 压力测试 | 1～2 道极简题；失败可触发「更简动画」 |
| 4 实践 | 间隔复测 / 绑定 | **v0.1 不做** |

### 5.1 默认路径（体验上的 C）
1. 用户输入或点选概念（首版：`循环`）。
2. 系统立即返回**分镜骨架**（3～7 个节拍：谁、做什么、矛盾/结果、对应原理点）。
3. 异步生成单文件 HTML 动画；写入工作区并右侧 iframe 播放。
4. 用户点「不懂 / 更简单」→ 以「降低要素、放慢、更直白」为约束再生成；可 `/重写`。
5. 短检验（界面题）→ 通过则结束本概念。

### 5.2 工程路径（抗慢）
- **骨架**：先返回结构化 JSON/Markdown 分镜，UI 立刻渲染。
- **缓存键**：`概念规范化 + 简版等级(0/1/2) + 协议版本`；命中则跳过生成。
- **金牌样例**：仓库内置「循环」已审动画，作缓存种子与质量锚点。
- **失败**：生成失败保留骨架 + 「再试一次」；不假装已理解。

## 6. 动画契约（v0.1）

- **形态**：自包含 **单文件 HTML**（内联 CSS/JS），无外网依赖（便于本地 iframe）。
- **内容约束**（写入生成 prompt，对齐母题）：
  - 物理/可感知悖论优先；找不到则诚实降级为「清楚类比」，不硬编。
  - 具体人物 + 动作推进；避免「系统/机制/维度」。
  - 5 岁用词标准；原理必须在**可见结果**里出现，不能只靠旁白念定义。
- **简版等级**：`L0` 标准；`L1/L2` 更少角色、更慢、更少并行信息。
- **非目标**：绘本级插画、配音、3D、游戏引擎。

## 7. 技术架构（薄壳）

### 7.1 建议栈
与 Tepeu 同族，降低搬运成本：
- **后端**：Java 21 + Spring Boot（可从 Tepeu 裁剪：chat stream、provider/key、工作区写文件）。
- **前端**：React + Vite；两栏布局；右侧 `iframe` sandbox 播动画。
- **教法**：内置 learn-skill 母题导师精简版（system prompt + 命令：`/重写` `/简单点`）。

### 7.2 从 Tepeu 借 / 不借

| 借 | 不借 |
|----|------|
| SSE 对话、Provider/API key 加密存储思路 | 终端 WebSocket |
| 工作区相对路径写 HTML + raw 预览 | 完整文件浏览器 / 技能市场 |
| （无）v0.1 不借 shell | 通用编程 Agent 工具环、`run_command` |

### 7.3 核心模块（逻辑）
1. **MotifSession** — 一轮概念的状态（骨架、动画路径、简版等级、检验结果）。
2. **StoryboardService** — 快速生成分镜骨架。
3. **AnimationGenerator** — 骨架 → HTML；写盘；更新缓存索引。
4. **DemoCache** — 按缓存键存/取动画文件。
5. **MotifTutor** — 母题对话编排（调用上述服务，穿插口诀与检验）。

## 8. v0.1 范围

### 做
- 两栏 UI + 母题对话最小闭环
- 骨架（短调用/本地）→ 动画 → 缓存 → 更简版（L0–L2）
- 动画后 **1 句口诀** + **1～2 道界面检验题**
- 金牌概念：**循环**（含内置 L0/L1/L2 种子）
- 本地配置一个 LLM Provider（简化版设置页）
- 命令：`/重写`、`/简单点`

### 不做
- 数理/科学内容库、多角色（数学陪练等）全开
- `/记忆卡`、间隔复测、账号、云同步
- 自动口播、复杂交互游戏、`run_command`
- 与 Tepeu 运行时插件化打通

## 9. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 现生成动画慢/翻车 | 骨架 + 缓存 + 金牌样例；失败可重试 |
| 故事与原理脱节 | prompt 强制「结果可见即原理」；人工审金牌样例 |
| 孩子乱点导致费用 | 同概念同等级缓存；简版次数上限（每概念最多升到 L2，共 3 档） |
| 范围膨胀回 IDE | 界面级禁止文件树/终端；Spec 红线 |
| 生成 HTML 含恶意脚本 | iframe `sandbox`（允许 scripts，禁止 top-navigation）；v0.1 不跑 shell |

## 10. 已拍板的歧义澄清（2026-07-12 复审）

1. **「不懂」与「更简单」**：同一动作——简版等级 +1（L0→L1→L2）并重新生成；文案可两个入口，语义相同。
2. **`/重写` vs 更简单**：`/重写` = 换故事场景（等级可不变）；「更简单」= 同场景降复杂度。v0.1 两个都要。
3. **金牌样例 ≠ 预置模板库主路径**：内置「循环」是缓存种子与质量锚点；新概念仍走生成，不以题库点选为主。
4. **3 秒骨架 SLA**：骨架走独立短调用（「循环」可用本地固定骨架）；动画走慢路径。禁止等整段 HTML 才首次刷新 UI。
5. **任意概念输入**：v0.1 允许输入，但只承诺「循环」质量；其它标「试做」。
6. **Step 2 口诀**：v0.1 必做——动画就绪后对话区 1 句口诀。
7. **检验**：v0.1 以界面 1～2 道选择题/判断题为准（可测）。
8. **命令**：v0.1 支持 `/重写`、`/简单点`；不做 `/记忆卡` 与间隔复测。
9. **`run_command`**：v0.1 不做。
10. **安全**：动画仅在 sandbox iframe 中播放。

## 11. 验证思路（v0.1）

- 单元：缓存键规范化、简版等级封顶、骨架 JSON 校验。
- 人工：金牌「循环」走通默认路径 + 更简单一次 + 检验通过。
- 回归：无 key / 生成失败时骨架或错误态可见，不白屏死等。
